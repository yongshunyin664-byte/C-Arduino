## 1. 静态局部变量（`static`）与普通局部变量的关键区别

### 一句话总结

> **作用域相同**（都在函数内部可见），但**生命周期不同**——`static` 变量只初始化一次，值在函数调用间保留，程序结束时才销毁；普通局部变量每次调用都重新分配，调用结束就消失。

### 对比表

| 对比维度 | 普通局部变量（auto） | 静态局部变量（`static`） |
|---------|---------------------|--------------------------|
| **存储位置** | 栈（stack） | 静态存储区（数据段） |
| **生命周期** | 函数调用期间，返回即销毁 | 程序整个运行期间 |
| **初始化** | 每次调用都需显式初始化，否则是垃圾值 | 只初始化**一次**，默认值为 0 |
| **值在调用间保留** | 不保留，每次都是新的 | 保留，下次调用沿用上次的值 |
| **作用域** | 函数内部可见 | 函数内部可见（二者相同） |

### 代码对比

```c
#include <stdio.h>

void demo() {
    int a = 0;          // 普通局部变量：每次调用都重置为 0
    static int b = 0;   // 静态局部变量：只在第一次调用时初始化为 0
    a++;
    b++;
    printf("a = %d, b = %d\n", a, b);
}

int main() {
    demo();  // a = 1, b = 1
    demo();  // a = 1, b = 2
    demo();  // a = 1, b = 3
    return 0;
}
```

- `a`（普通局部变量）：每次调用被重置为 0 → 始终输出 1
- `b`（静态局部变量）：只初始化一次，值持续累加 → 依次输出 1、2、3

### 常见考点

1. **默认值**：`static` 局部变量若未显式初始化，默认值为 **0**；普通局部变量则是随机垃圾值。
2. **初始化时机**：`static` 局部变量的初始化在**编译时**确定，在 `main()` 执行前已完成。
3. **适用场景**：需要记录函数被调用的次数、保留上一次计算状态等场景。

## 2. `static` 全局变量、普通全局变量、局部变量的作用域

### 一句话总结

> **`static` 修饰全局变量 → 缩小作用域**（整个程序 → 仅本文件），防止其他文件用 `extern` 访问。
> **`static` 修饰局部变量 → 延长生命周期**（函数调用期间 → 程序运行期间），但作用域不变。

### 变量分类图

```
                    作用域
                    /    \
            文件级         函数级
            /    \           /    \
    static全局   普通全局  普通局部  static局部
   (仅本文件)  (整个程序)  (仅本函数)(仅本函数)
```

### 对比表

| 变量类型          | 作用域          | 生命周期     | 其他文件能访问？ |
|------------------|-----------------|-------------|:---:|
| 普通全局变量      | 整个程序         | 程序运行期间 | ✅（`extern` 声明后） |
| `static` 全局变量 | **仅本文件内**  | 程序运行期间 | ❌ |
| 普通局部变量      | 仅函数内         | 函数调用期间 | ❌ |
| `static` 局部变量 | 仅函数内         | 程序运行期间 | ❌ |

### 代码示例

```c
// file1.c
int global_a = 10;          // 普通全局变量：整个程序可访问
static int global_b = 20;   // static 全局变量：仅 file1.c 内可访问
                            // 其他文件无法 extern 它

void func() {
    int local_c = 30;       // 普通局部变量：仅 func 内有效
    static int local_d = 0; // static 局部变量：仅 func 内有效，但值跨调用保留
}
```

```c
// file2.c
extern int global_a;  // ✅ OK，可以引用 file1.c 的普通全局变量
extern int global_b;  // ❌ 链接错误！static 全局变量对 file2.c 不可见
```

### `static` 的双重含义（重要）

| `static` 修饰位置 | 改变的是什么          | 效果 |
|------------------|----------------------|------|
| 修饰**全局变量** | **作用域**（链接属性） | 整个程序 → 仅本文件（internal linkage） |
| 修饰**局部变量** | **生命周期**          | 函数调用期间 → 程序运行期间（作用域不变） |
| 修饰**函数**     | **作用域**            | 整个程序 → 仅本文件（与全局变量同理） |

## 3. `const` 关键字：只读变量

### 一句话总结

> `const` = **只读**。"我承诺不再改这个东西"，编译器帮你看住。

### 基础用法

```c
const int a = 10;  // 声明只读变量，赋初值
a = 20;            // ❌ 编译报错！不许改
```

### `const` 不是真正的常量

| | `const int a = 10` | `#define PI 3.14` |
|--|--|--|
| 是什么 | 一个"不许改"的变量 | 纯文本替换 |
| 有地址吗 | ✅ `&a` 可以取到 | ❌ 没有地址 |
| 占内存吗 | ✅ | ❌ |
| 有类型检查 | ✅ | ❌ |

> **`const` 变量 = 只读变量，不是常量。** 真正的常量是 `#define` 或枚举。

### `const` 与指针搭配（重点）

**万能公式：`const` 在 `*` 左 → 锁内容；`const` 在 `*` 右 → 锁指针。**

| 写法 | 读法 | 哪个不能改 |
|------|------|:---:|
| `const int *p` | "p 指向一个只读 int" | `*p`（内容） |
| `int const *p` | 同上（`const` 紧挨 `int`） | `*p`（内容） |
| `int * const p` | "p 是一个只读指针，指向 int" | `p`（指针本身） |
| `const int * const p` | 双重只读 | 内容和指针都不能改 |

### 代码验证

```c
int x = 10, y = 20;

// const 在 * 左边 → 锁内容
const int *p1 = &x;
*p1 = 100;   // ❌ 内容锁了
p1 = &y;     // ✅ 指针没锁，可以指向别处

// const 在 * 右边 → 锁指针
int * const p2 = &x;
*p2 = 100;   // ✅ 内容没锁，可以改
p2 = &y;     // ❌ 指针锁了，不能指向别处

// 两边都锁
const int * const p3 = &x;
*p3 = 100;   // ❌
p3 = &y;     // ❌
```

### 本质理解

> **`const` 修饰它左边的东西。左边没有就修饰右边的。**

```c
int const *p;   // const 左边是 int → 锁的是内容
int * const p;  // const 左边是 *   → 锁的是指针
const int *p;   // const 左边没东西 → 锁右边的 int（内容）
```

### 记忆口诀

> **左内容，右指针。** `const` 在 `*` 的左边 → 锁内容；在 `*` 的右边 → 锁指针。
