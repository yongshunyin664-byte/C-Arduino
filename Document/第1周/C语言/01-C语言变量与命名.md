# C语言变量与变量命名

---

## 一、什么是变量

变量是程序中用于存储数据的**命名内存空间**。可以把它想象成一个带标签的盒子——标签是变量名，盒子里装的是数据。

```c
int age = 25;      // age 是变量名，25 是存在里面的值
```

每个变量在内存中占据一块空间，变量名就是这块空间的别名。通过变量名，我们可以方便地读写内存中的数据。

---

## 二、变量的声明与初始化

```c
// 1. 先声明，后赋值
int count;
count = 10;

// 2. 声明的同时初始化（推荐做法）
int count = 10;

// 3. 同时声明多个同类型变量
int a, b, c;
int x = 1, y = 2, z = 3;
```

> **建议**：始终在声明变量时进行初始化。未初始化的变量包含随机值（垃圾值），使用前如不赋值可能导致不可预期的行为。

## 三、C语言的基本数据类型

| 类型         | 关键字      | 占用字节 | 示例                        |
|--------------|-------------|----------|---------------------------|
| 字符型       | `char`      | 1        | `char ch = 'A';`          |
| 短整型       | `short`     | 2        | `short s = 100;`          |
| 整型         | `int`       | 4        | `int a = 1000;`           |
| 长整型       | `long`      | 4 或 8   | `long l = 100000L;`       |
| 长长整型     | `long long` | 8        | `long long ll = 99999LL;` |
| 单精度浮点型 | `float`     | 4        | `float f = 3.14f;`        |
| 双精度浮点型 | `double`    | 8        | `double d = 3.1415926;`   |

> 注意：`float` 类型的字面量需加 `f` 后缀，否则默认被视为 `double`。

## 四、变量命名规则（必须遵守）

### 4.1 合法规则

1. **只能由字母（A-Z, a-z）、数字（0-9）和下划线（`_`）组成**
2. **第一个字符不能是数字**，必须是字母或下划线
3. **严格区分大小写**——`age`、`Age`、`AGE` 是三个互不相关的变量
4. **不能使用C语言的关键字（保留字）作为变量名**

### 4.2 示例

```c
// ✅ 合法的变量名
int age;
int _count;
int player1;
int total_score;
int MAX_SIZE;
int Über;           // C99起支持Unicode字符（但不推荐）

// ❌ 非法的变量名
int 1st_place;      // 数字开头
int my-variable;     // 包含连字符 -
int float;           // float 是关键字
int user name;       // 包含空格
int $price;          // 包含美元符号 $
int user@email;      // 包含 @ 符号
```

## 六、命名风格（惯例/建议）

虽然不是编译器强制的，但良好的命名风格能极大提升代码可读性和可维护性。

### 6.1 常见命名风格

| 风格                           | 示例                                    | 常用场景              |
|--------------------------------|-----------------------------------------|-----------------------|
| **蛇形命名** (snake_case)      | `user_name`, `max_count`, `total_score` | C语言中最常见、最推荐  |
| **驼峰命名** (camelCase)       | `userName`, `maxCount`, `totalScore`    | 部分C项目使用         |
| **帕斯卡命名** (PascalCase)    | `UserName`, `MaxCount`                  | 结构体/类型名常用     |
| **全大写下划线** (UPPER_SNAKE) | `MAX_SIZE`, `PI`, `BUFFER_LEN`          | 宏定义、常量          |
| **下划线前缀**                 | `_private_var`, `_internal_func`        | 表示"私有/内部"       |

### 6.2 命名示例

```c
// ✅ 好的命名——见名知意
int student_count = 30;
float average_score = 85.5;
char user_choice = 'y';
double circle_radius = 5.0;

// ✅ 宏常量的命名
#define MAX_BUFFER_SIZE 1024
#define PI 3.1415926
#define DEFAULT_PORT 8080

// ❌ 差的命名——含义不清
int sc = 30;          // sc 是什么？student_count 还是 score？
float as = 85.5;      // as 像英文单词，不是变量名
char uc = 'y';        // uc 意义不明

// ❌ 差的命名——过度缩写
int cnt = 0;          // 不如写成 count
int usr_id = 1;       // 不如写成 user_id
void proc_data();     // 不如写成 process_data

// ❌ 差的命名——拼写错误或拼音
int stduent = 10;     // 拼写错误：应为 student
int xuesheng = 10;    // 拼音：不要用拼音
```

## 七、变量的作用域与生命周期

```c
#include <stdio.h>

int global_var = 100;           // 全局变量——整个文件可用，程序结束后销毁

static int file_var = 50;       // 文件作用域——仅本文件内可用

int main() {
    int local_var = 10;         // 局部变量——仅在 main() 内可用
    static int static_var = 5;  // 静态局部变量——值在函数调用间保持

    {
        int block_var = 99;     // 块作用域——仅在该 {} 块内可用
        printf("%d\n", block_var);
    }
    // block_var 在此处不可访问

    return 0;
}
```

| 存储类别          | 作用域    | 生命周期    | 默认初始值 |
|------------------|-----------|------------|-----------|
| 局部变量 `auto`   | 函数/块内 | 函数调用期间 | 垃圾值    |
| 全局变量          | 整个程序  | 整个程序     | 0        |
| 静态局部 `static` | 函数内    | 整个程序     | 0        |
| 静态全局 `static` | 本文件    | 整个程序     | 0        |
| `register`       | 函数内    | 函数调用期间  | 垃圾值   |

## 八、变量使用的常见误区

### 8.1 未初始化即使用

```c
int x;              // 未初始化
printf("%d", x);    // 未定义行为！x 是垃圾值
```

### 8.2 命名与关键字冲突

```c
int float = 5;      // 编译错误
int class = 10;     // C中合法，但C++中是关键字
```

### 8.3 变量遮蔽（Shadowing）

```c
int x = 10;
{
    int x = 20;     // 遮蔽外部的 x
    printf("%d", x); // 输出 20
}
printf("%d", x);     // 输出 10（外部 x 未变）
```

## 九、升级版速查总结

| 维度           | 要点                                                                |
|----------------|---------------------------------------------------------------------|
| **语法合法性** | 字母/数字/下划线，不能数字开头，不能用关键字 |
| **拼写正确性** | 无拼写错误，无拼音，无过度缩写 |
| **命名风格**   | snake_case 用于变量，UPPER_SNAKE 用于常量/宏，PascalCase 用于结构体名 |
| **类型选择**   | 整数→`int`，小数→`double`，字符→`char` |
| **初始化**     | 声明时立即初始化，指针初始化为 `NULL` |
| **const 保护** | 不应被修改的变量加 `const` |
| **无魔数**     | 直接出现的数字替换为有名称的常量/宏 `#define` |
| **作用域**     | 最小化作用域，避免无意义的全局变量，循环变量定义在循环内 |
| **类型安全**   | 避免隐式转换；整数除法、赋值截断要显式处理 |
| **无冗余**     | 删掉声明但从未使用的变量 |

## 十、一键复制：打码后让 AI 检查的话术

打完代码后，把下面这段话粘贴给 AI：

> 帮我检查代码的变量问题，按这 10 项逐条检查：
> 1. 变量声明后是否都已被使用（无冗余变量）
> 2. 数据类型选择是否合适、类型匹配是否正确
> 3. 变量是否均已初始化、初始值是否合理、指针是否初始化为 NULL
> 4. 变量命名是否合法（字母数字下划线、不能数字开头、不能是关键字）
> 5. 变量拼写是否有误
> 6. 变量命名风格是否符合场景（snake_case / UPPER_SNAKE）
> 7. 是否该用 const 保护不应修改的变量
> 8. 是否有魔数，应替换为 #define 常量/宏
> 9. 隐式类型转换是否有风险（整数除法、赋值截断等）
> 10. 变量作用域是否足够小、是否有不必要的全局变量
