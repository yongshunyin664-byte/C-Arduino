### switch-case

> ❌ 你的答案：不知道 → 正确答案：**C（float）**
> 📌 `switch` 只能用**整型表达式**（int/char/枚举），不能是 float/double 浮点类型。

**36.** 以下代码中，若 `grade = 2`，输出结果是：
```c

    case 1: printf("A"); break;
    case 2: printf("B"); break;
    case 3: printf("C"); break;
    default: printf("?");
}
```
A. `A`
B. `B`
C. `ABC?`
D. `B?`

> ✅ 你的答案：**B**

**37.** 如果在 `case 2` 后面忘记写 `break`，会发生什么？
```c

    case 1: printf("A"); break;
    case 2: printf("B");
    case 3: printf("C"); break;
    default: printf("?");
}
// grade = 2
```
A. 只输出 `B`
B. 输出 `BC`（穿透 fall-through：执行完 case 2 后继续执行 case 3）
C. 编译报错
D. 输出 `B?`

> ✅ 你的答案：**C** → ❌ 正确答案：**B（输出 BC）**
> 📌 忘记 break 会导致**穿透（fall-through）**：case 2 执行完后不退出，继续执行 case 3 的代码，输出 BC。

**38.** `switch` 语句中 `default` 的作用是：

**39.** 以下关于 `switch-case` 和 `if-else if` 的说法，**错误**的是：

> 📌 D 说法正确（default 是可选的），C 说法**错误**：break 不是必须的，有时故意不加 break 来实现穿透效果。

## 四、简答题（每题1分，共10分）

**116.** 简述 `=` 和 `==` 的区别。在 `if` 语句中误用会有什么后果？请举例说明。

> ✅ 你的答案：`=` 是赋值，`==` 是逻辑运算符用于判断。错误使用时 `==` 变成赋值，`=` 变成判断。
>
> 📌 **补充**：`if (x = 5)` 是赋值，永远为真，不报错；`if (x == 5)` 是比较。这是最常见的 C 语言陷阱之一。

**117.** 简述逻辑运算符 `&&` 和 `||` 的短路特性。这种特性有什么实际用途？

> ✅ 你的答案：`&&` 优先级高于 `||`。`&&` 是逻辑与，必须两边条件都为 1（真）才成立；`||` 是逻辑或，两边只有一个为真即可。
>
> ⚠️ 你回答了优先级和含义，但**未答短路特性**。📌 **正确答案**：`a && b` 中 a 为假则 b 不执行；`a || b` 中 a 为真则 b 不执行。实际用途：如 `if (p != NULL && p->value > 0)` 可防止空指针访问。

**118.** 简述 `while` 循环和 `do-while` 循环的核心区别。分别适合什么场景？

> ✅ 你的答案：while 判断为真才运行循环体，do-while 先运行一次再判断条件。
>
> 📌 while 适合"可能零次"的场景（如不确定是否满足条件）；do-while 适合"至少一次"的场景（如菜单至少显示一次）。
